## 为什么单独列出

workspace 根清单除了 73 个 `codegen/common` 一方 crate，还包含 build helper、`prod` 目录和 vendored third-party crate。它们同样参与编译，但来源、发布对象和架构意义不同。把所有成员直接算成“Grok Build 产品模块”会造成错误结论。

## 分类规则

- `crates/build` 在编译期生成代码，不属于用户请求运行时。
- `prod` 表达单体仓库内的生产服务或共享类型；公开仓库并不证明对应服务的完整线上实现。
- `third_party` 保存外部来源或移植代码，应以各自来源和许可说明为准。
- `crates/codegen` 与 `crates/common` 构成本白皮书的 73 个一方 crate 主图谱。

深度文章可以解释一方封装如何调用第三方布局或协议库，但不会把第三方算法改写成 xAI 的原创设计。源码摘录继续保留上游 Apache-2.0 和各第三方目录中的原始来源信息。
